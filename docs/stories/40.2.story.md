## Story

As a user viewing the insights dashboard,
I want each stats section wrapped in styled bordered panels with a structured layout,
So that the dashboard feels polished and organized rather than raw text.

## Background

The current insights view renders stats as plain text with manual spacing and `fmt.Fprintf` alignment. Lipgloss provides `NewStyle().Border()`, `JoinHorizontal()`, and `JoinVertical()` for structured layouts with borders. The dashboard should use a 2-column grid where possible, with themed header colors.

## Acceptance Criteria

**Given** the insights view is displayed
**When** each stats section (Completion Trends, Streaks, Mood & Productivity, Door Preferences) renders
**Then** each section is wrapped in a Lipgloss bordered panel using `RoundedBorder()` or similar
**And** each panel has a styled header/title line
**And** panels are arranged in a 2-column layout using `JoinHorizontal()` where terminal width permits

**Given** the terminal width is narrow (< 80 columns)
**When** the insights view renders
**Then** panels stack vertically in a single column (responsive fallback)

**Given** the insights view is displayed with the viewport/scrolling system
**When** content exceeds terminal height
**Then** the existing viewport scrolling behavior is preserved with the new panel layout

**Given** the current insights view data
**When** the styled dashboard renders
**Then** all existing data is still displayed (no data loss from the layout change)

## Technical Notes

- Use `lipgloss.NewStyle().Border(lipgloss.RoundedBorder()).BorderForeground(color)`
- Use `lipgloss.JoinHorizontal(lipgloss.Top, leftPanel, rightPanel)` for 2-column
- Measure terminal width to decide between 2-column and single-column layout
- Header styles should use foreground colors, not background (readability across themes)
