## Story

As a user who has selected a door theme,
I want the stats dashboard colors to match my chosen theme,
So that the entire application feels visually cohesive.

## Background

Door themes (Modern, Classic, Sci-Fi, Shoji, Golden, Shadow) each define a `ThemeColors` struct with door-specific colors. The stats dashboard (from Stories 40.1-40.7) uses its own hardcoded colors. This story extends `ThemeColors` to include a stats color palette so each theme defines how sparklines, bars, panels, and heatmaps look.

## Acceptance Criteria

**Given** the user has selected a door theme
**When** the insights dashboard renders
**Then** sparkline gradient colors match the theme palette
**And** bar chart fill colors match the theme palette
**And** panel border colors match the theme palette
**And** the overall visual impression is cohesive with the doors view

**Given** each existing theme (Classic, Modern, Sci-Fi, Shoji, Golden, Shadow)
**When** stats are rendered
**Then** each theme produces a visually distinct stats appearance
**And** all themes maintain sufficient contrast for readability

**Given** a new theme is added in the future
**When** it does not define stats colors
**Then** a sensible default stats palette is used (no crash, no missing colors)

## Technical Notes

- Extend `ThemeColors` with stats-specific fields (sparkline gradient endpoints, bar fill, panel border, etc.)
- Each theme implementation provides its palette (e.g., Sci-Fi = neon greens/blues, Golden = warm amber)
- Stats rendering functions accept theme colors as parameter instead of hardcoded values
- Default palette for themes that don't define stats colors
