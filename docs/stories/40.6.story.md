## Story

As a user opening the insights view,
I want numbers to count up from 0 to their final value with a brief animation,
So that the stats reveal feels satisfying and deliberate — like a dashboard powering on.

## Background

Currently, stats appear instantly as static text. Bubbletea supports animation via `tea.Tick`. A ~500ms counting animation on key stats (tasks completed, streak days, total doors) creates "button feel" polish aligned with SOUL.md's "every interaction should feel deliberate."

## Acceptance Criteria

**Given** the user navigates to the insights view
**When** the view first opens
**Then** numeric stats (tasks today, streak, total completions) animate from 0 to their final values
**And** the animation completes within ~500ms (not sluggish)
**And** the animation uses `tea.Tick` for frame updates

**Given** the animation is in progress
**When** the user presses Esc or any navigation key
**Then** the animation stops immediately and final values display instantly (no blocking)

**Given** the user returns to the insights view after leaving
**When** the view reopens
**Then** the animation plays again (resets on re-entry)

**Given** a stat value is 0
**When** the animation runs
**Then** it skips animation for that stat and displays 0 immediately

**Given** the sparkline from Story 40.1
**When** the insights view opens
**Then** the sparkline optionally draws left-to-right (bars appear sequentially)
**And** this is non-blocking and interruptible

## Technical Notes

- Add animation state to the insights view model (`animating bool`, `animProgress float64`, `animStartTime`)
- Use `tea.Tick(33*time.Millisecond, ...)` for ~30fps updates
- Calculate displayed value as `int(float64(finalValue) * progress)` where progress goes 0→1
- Easing function (ease-out) makes the counting feel natural (fast start, slow finish)
- Animation must not block Update() — it's purely a View() concern driven by tick messages
