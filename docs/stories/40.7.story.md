## Story

As a user viewing my insights,
I want to see a "Session Highlights" section surfacing data that is tracked but currently hidden,
So that I can discover interesting patterns about my own behavior.

## Background

ThreeDoors tracks extensive data in sessions.jsonl that is never shown to the user:
- Time to first door (engagement speed)
- Detail view count (curiosity metric)
- Notes added count (engagement metric)
- Session duration (how long sessions last)
- Streak history (longest streak, average streak)

These are all interesting personal observations that align with the SOUL.md "hey, look at what you did!" tone.

## Acceptance Criteria

**Given** the user has session history
**When** the insights view renders
**Then** a "Session Highlights" section displays:
  - Average session duration
  - Fastest time to first door selection
  - Total doors opened (all time)
  - Total detail views opened (curiosity metric)
  - Streak stats: current, longest ever, average streak length

**Given** the streak stats are displayed
**When** the current streak equals or exceeds the longest streak
**Then** the display notes this is a personal best (celebratory, not pressuring)

**Given** minimal session data (< 3 sessions)
**When** the highlights section renders
**Then** metrics that require more data show "—" or a friendly "keep going" message
**And** available metrics still display

**Given** the SOUL.md alignment requirement
**When** highlights are displayed
**Then** all language is observational and celebratory
**And** no metric is framed as "you should improve this"
**And** no negative comparisons are made

## Technical Notes

- Session duration: `end_time - start_time` from sessions.jsonl
- Time to first door: `time_to_first_door_seconds` field already captured
- Streak history: compute from daily completion dates; may already exist in PatternAnalyzer
- Display in a bordered panel (consistent with Story 40.2 dashboard layout)
