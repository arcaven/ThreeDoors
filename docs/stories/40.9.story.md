## Story

As a user who crosses a meaningful usage threshold,
I want a brief, one-time celebration banner in the insights view,
So that I feel recognized for what I've accomplished without any pressure to achieve more.

## Background

CAUTION: This story is closest to the SOUL.md "no gamification" boundary. The key distinction: milestones celebrate what happened organically, not what the user should do. No "only 3 more tasks to reach gold level!" language. No badges, no levels, no unlockables. One-time display only.

Example milestones:
- 100 doors opened: "Century Club! You've opened 100 doors."
- 50 tasks completed: "Half-century! 50 tasks done."
- 10-day streak: "Double digits! 10 days in a row."
- First session: "Welcome! Your journey starts here."

## Acceptance Criteria

**Given** the user crosses a milestone threshold during a session
**When** they next view the insights dashboard
**Then** a styled celebration banner appears at the top of the view
**And** the banner uses celebratory language that observes what happened
**And** the banner does NOT suggest what to do next or set future targets

**Given** a milestone has been displayed once
**When** the user views insights again
**Then** the same milestone is NOT shown again (persisted "seen" state)

**Given** the milestone display
**When** all milestone text is reviewed
**Then** no milestone uses language like "keep going to reach...", "next milestone...", or "only X more..."
**And** every milestone celebrates a past achievement, never prescribes future behavior

**Given** the milestone persistence
**When** milestones are stored
**Then** seen milestones are persisted to `~/.threedoors/milestones.json` or equivalent
**And** the file uses atomic writes (consistent with project patterns)

**Given** the user has never used ThreeDoors before
**When** they complete their first session
**Then** a "Welcome" milestone is available on next insights view

## Technical Notes

- Define milestones as a data structure: threshold, metric type, message template
- Store seen milestones in a small JSON file
- Check thresholds on insights view entry, not on every session (performance)
- Banner styling: bordered panel with accent color, displayed above other stats panels
