## Story

As a user viewing my activity patterns,
I want a GitHub-style activity heatmap showing daily completion density over recent weeks,
So that I can see patterns at a glance — which days I'm active and consistency over time.

## Background

The current daily stats are a plain text table. A 7-row (days of week) x 8-column (weeks) grid using colored Unicode blocks would be instantly recognizable and visually striking. Each cell's color intensity represents completion count.

Before implementing, evaluate `NimbleMarkets/ntcharts` heatmap component vs building in-tree. Record the dependency decision in BOARD.md.

## Acceptance Criteria

**Given** 8 weeks of daily completion data
**When** the activity heatmap renders
**Then** a 7×8 grid displays with rows = days of week (Mon–Sun) and columns = weeks
**And** each cell uses color intensity to represent completion count
**And** a legend maps colors to completion ranges (e.g., 0, 1-2, 3-4, 5+)
**And** the current day is visually identifiable

**Given** fewer than 8 weeks of data
**When** the heatmap renders
**Then** only available weeks are shown (no empty columns for future dates)

**Given** a day with zero completions
**When** that cell renders
**Then** it displays as empty/blank (neutral — no guilt-inducing styling)

**Given** the SOUL.md alignment requirement
**When** the heatmap is displayed
**Then** empty days are visually neutral (not red, not crossed out)
**And** no streak-breaking or gap-highlighting is applied

**Given** the dependency evaluation
**When** implementation begins
**Then** a decision is recorded in BOARD.md: ntcharts heatmap vs in-tree implementation
**And** the rationale includes: API fit, dependency size, maintenance risk

## Technical Notes

- Color scale: empty = no shading, light green = 1-2, medium green = 3-4, bright green = 5+
- Use Lipgloss `AdaptiveColor` for light/dark terminal support
- If using ntcharts: `ntcharts.NewHeatmap()` with Lipgloss styling
- If in-tree: render grid using `fmt.Fprintf` with per-cell Lipgloss styling
- Aggregate daily data from sessions.jsonl using existing session reader
