## Story

As a user viewing my completion trends,
I want the sparkline to use color gradients from cool (low) to warm (high),
So that the visualization is visually delightful and instantly communicates relative performance.

## Background

The current `sparkline()` function in `insights_view.go` returns plain Unicode block characters (`sparkChars`) in monochrome. Lipgloss `Blend1D()` is already available in the project's current version and can produce smooth color gradients. Zero completions should render as dim gray; high completions as bright green/gold.

## Acceptance Criteria

**Given** the insights view with 7-day completion data
**When** the sparkline is rendered
**Then** each bar character is individually colored using a gradient scale
**And** the lowest value renders in a cool/dim color (e.g., dim gray or muted blue)
**And** the highest value renders in a warm/bright color (e.g., bright green or gold)
**And** intermediate values are smoothly interpolated between the endpoints
**And** the gradient uses `lipgloss.Blend1D()` or equivalent Lipgloss color function

**Given** a terminal with limited color support (e.g., 16 colors)
**When** the sparkline renders
**Then** it degrades gracefully (falls back to the current monochrome behavior or uses available colors)

**Given** all completion values are zero
**When** the sparkline renders
**Then** all bars render in the dim/lowest color (no division-by-zero or NaN in gradient calculation)

**Given** all completion values are equal (non-zero)
**When** the sparkline renders
**Then** all bars render in a uniform mid-range color (gradient handles flat data)

## Technical Notes

- Modify `sparkline()` to accept or return styled strings instead of raw runes
- Use `lipgloss.Blend1D(colorA, colorB, ratio)` where ratio = value/maxValue
- Consider `lipgloss.AdaptiveColor` for light/dark terminal compatibility
- No new dependencies required
