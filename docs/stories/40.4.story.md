## Story

As a user viewing mood and productivity correlations,
I want proportional horizontal bar charts with color-coded fills,
So that I can instantly compare how different moods and times of day affect my task completion.

## Background

The current mood correlation display is plain text: "Focused: avg 4.2 tasks (12 sessions)". Horizontal bars make comparison visual and instant. Each mood gets a distinct color. A compact 24-hour activity bar shows when the user is most active.

## Acceptance Criteria

**Given** the insights view with mood correlation data
**When** the mood section renders
**Then** each mood displays a proportional horizontal bar filled relative to the highest average
**And** each mood bar has a distinct color (e.g., Focused=blue, Energized=yellow, Calm=green, Tired=gray)
**And** the numeric average and session count are shown alongside the bar
**And** bars are aligned vertically for easy comparison

**Given** time-of-day session data
**When** the activity section renders
**Then** a compact 24-hour bar shows activity density per time block
**And** peak activity periods are labeled (e.g., "Peak: 9am-11am")
**And** the visualization uses Unicode block characters with color intensity

**Given** no mood data has been recorded
**When** the mood bar chart would render
**Then** a placeholder message is shown instead of empty bars

**Given** only one mood has been recorded
**When** the bar chart renders
**Then** the single bar fills to 100% and renders correctly

## Technical Notes

- Bar rendering: fill character (e.g., `█` or `=`) repeated proportionally to max width
- Use Lipgloss foreground coloring per mood
- Time-of-day data available from MCP PatternMiner; may need to expose in core
- Keep bar chart rendering as a simple helper function, not a separate package
