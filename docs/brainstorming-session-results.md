## Idea Categorization

### Immediate Opportunities
*Ideas ready to implement in MVP*

1. **Simple progress over perfection mindset**
   - Description: Core UX philosophy - encourage starting, allow imperfect action, celebrate movement
   - Why immediate: This is THE defining feature, the north star for all design decisions
   - Resources needed: UX design thinking, thoughtful messaging/copy

2. **Choose-your-own-adventure task flow**
   - Description: Navigate options and manage emotional state through interactive choices
   - Why immediate: Embodies user agency and adaptive partnership
   - Resources needed: Go interaction model, decision tree logic

3. **Local storage (Apple Notes or simple file-based)**
   - Description: Dead simple task capture with no friction - local JSON or Apple Notes integration
   - Why immediate: Removes external dependencies, keeps it simple for BMAD demo
   - Resources needed: Go file I/O or macOS scripting bridge

4. **One improvement prompt per session**
   - Description: "What's one thing you could improve about this list/task/goal right now?"
   - Why immediate: Forces continuous improvement without overwhelming user
   - Resources needed: Simple prompt system, one question per session

5. **Yesterday comparison**
   - Description: Show you did something better than yesterday (any dimension of improvement)
   - Why immediate: Makes progress visible and measurable
   - Resources needed: Simple state persistence, delta calculation

6. **Values/goals always visible**
   - Description: Keep big picture in sight during task work
   - Why immediate: Maintains alignment, prevents getting lost in minutiae
   - Resources needed: Static display area, user-defined values input

### Future Innovations
*Ideas requiring development/research*

1. **Trading mechanic - easy tasks unlock rewards**
   - Description: Offer simple micro-tasks that progress important work and unlock rewards
   - Development needed: Reward system design, task dependency mapping, unlock mechanics
   - Timeline estimate: MVP+1 (2-4 weeks after MVP)

2. **LLM-powered assumption challenging**
   - Description: AI questions your dependencies, finds shortcuts, collapses chains
   - Development needed: LLM integration (OpenAI/Anthropic API), prompt engineering, context management
   - Timeline estimate: MVP+1 (2-4 weeks after MVP)

3. **Jira import + Calendar/Reminder integration**
   - Description: Pull work tasks from Jira, sync with system calendar/reminders
   - Development needed: API integrations, data mapping, sync strategies
   - Timeline estimate: MVP+2 (4-8 weeks after MVP)

4. **Basic replanning capabilities**
   - Description: Handle changes gracefully - manual triggers, simple rescheduling
   - Development needed: Conflict detection, rescheduling algorithms, user interaction design
   - Timeline estimate: MVP+2 (4-8 weeks after MVP)

5. **Task emotional weighting system**
   - Description: App learns which tasks cause avoidance, stress; auto-weights over time
   - Development needed: Behavioral tracking, ML model (lightweight), pattern recognition
   - Timeline estimate: MVP+3 (8-12 weeks after MVP)

6. **Habit-based gentle check-ins**
   - Description: Low-pressure re-engagement, no guilt, work with resistance
   - Development needed: Notification system, timing intelligence, tone/copy development
   - Timeline estimate: MVP+3 (8-12 weeks after MVP)

- **Humans exist in states, not static conditions**: Minute-by-minute changes in energy, emotion, capacity. App must adapt to human reality

- **Intersection of intention and action**: Todo apps are uniquely positioned to unlock human potential because they live where decisions become reality
